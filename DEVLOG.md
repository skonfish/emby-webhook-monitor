
# 开发日志

## 2024-05-20: V0.1 版本上线准备

**目标**: 将原型整理为可发布、可部署的项目。

**完成工作**:
1.  **架构分离**: 确定了 前端 (React/Vite) + 后端 (Express) 的单体容器架构。前端构建后由后端静态托管，方便部署。
2.  **数据存储**: 选择了 `lowdb`。原因：需求仅为简单的日志记录，不需要 MySQL/Mongo 这样重的数据库，JSON 文件存储配合 Docker Volume 完美符合需求。
3.  **Docker化**: 编写了多阶段构建的 Dockerfile。
    - Stage 1: Node 环境构建 React 前端。
    - Stage 2: 拷贝构建产物 (`dist`) 到后端目录，安装仅生产环境依赖。
4.  **容错处理**: 
    - 发现 Emby 的 Webhook 数据结构在不同版本可能有细微差别，采用了可选链 (`?.`) 进行安全访问。
    - 针对 `ip` 字段，目前暂时使用模拟数据填充地理位置，后续版本计划引入 `geoip-lite`。

**遇到的问题与解决方案**:
- *问题*: Docker 容器启动时报错找不到 `db` 目录。
- *解决*: 在 `server.js` 启动前增加了 `fs.existsSync` 检查，不存在则自动 `mkdir`。

**下一步计划 (V0.2)**:
- 引入真实的 IP 地理位置库。
- 增加简单的 Basic Auth 认证，防止外网直接访问面板。
- 优化移动端表格的显示体验。
